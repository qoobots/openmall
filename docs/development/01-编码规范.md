## 2. 命名规范

### 2.1 通用命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | 大驼峰（PascalCase） | `UserService`, `ProductController` |
| 方法名 | 小驼峰（camelCase） | `getUserById`, `createOrder` |
| 变量名 | 小驼峰（camelCase） | `userName`, `orderStatus` |
| 常量名 | 全大写下划线 | `USER_SESSION_KEY`, `MAX_RETRY_COUNT` |
| 包名 | 全小写点分隔 | `com.qoobot.openmall.portal.controller` |

### 2.2 数据库命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线，单数 | `user`, `product_spu` |
| 字段名 | 小写下划线 | `user_id`, `order_status` |
| 主键 | `id` | `id` |
| 外键 | 关联表名_字段名 | `user_id`, `category_id` |
| 索引 | uk_/idx_ 前缀 | `uk_username`, `idx_user_type` |

### 2.3 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| Java 类文件 | 与类名一致 | `UserController.java` |
| HTML 模板 | 小写中划线 | `product-list.html`, `user-profile.html` |
| CSS 文件 | 小写中划线 | `admin-style.css`, `shop-theme.css` |
| JS 文件 | 小写中划线 | `cart-utils.js`, `order-form.js` |
| 配置文件 | 小写点分隔 | `application-dev.yml` |

## 3. 分层编码规范

### 3.1 Controller 层

```java
@Controller
@RequestMapping("/products")
public class ProductController {

    private final ProductService productService;

    public ProductController(ProductService productService) {
        this.productService = productService;
    }

    // 列表页：返回模板视图名
    @GetMapping
    public String list(Model model,
                       @RequestParam(defaultValue = "1") int page,
                       @RequestParam(defaultValue = "10") int size) {
        PageResult<ProductVO> result = productService.findPage(page, size);
        model.addAttribute("page", result);
        return "product/list";
    }

    // 详情页：返回模板视图名
    @GetMapping("/{id}")
    public String detail(@PathVariable Long id, Model model) {
        ProductVO product = productService.findById(id);
        model.addAttribute("product", product);
        return "product/detail";
    }

    // AJAX 接口：返回 JSON
    @PostMapping("/{id}/favorite")
    @ResponseBody
    public Result<Void> favorite(@PathVariable Long id) {
        productService.addFavorite(id);
        return Result.success();
    }
}
```

**规范要点**：
- 使用构造器注入（不使用 `@Autowired` 字段注入）
- 返回视图名的方法不加 `@ResponseBody`
- 返回 JSON 的方法加 `@ResponseBody` 或使用 `@RestController`
- 方法命名语义清晰：`list`、`detail`、`create`、`update`、`delete`
- 分页参数提供默认值

**规范要点**：
- 接口与实现分离

### 3.3 Repository 层

```java
@Repository
public interface ProductSpuRepository extends JpaRepository<ProductSpu, Long> {

    // 简单查询：方法命名自动推导
    List<ProductSpu> findByShopId(Long shopId);

    // 分页查询
    Page<ProductSpu> findByIsOnSaleAndIsDeleted(Integer isOnSale,
                                                  Integer isDeleted,
                                                  Pageable pageable);

    // 带条件的查询
    List<ProductSpu> findByCategoryIdAndIsOnSale(Long categoryId,
                                                   Integer isOnSale);

    // 自定义 JPQL 查询
    @Query("SELECT p FROM ProductSpu p WHERE p.shopId = :shopId " +
           "AND p.isOnSale = 1 AND p.isDeleted = 0")
    List<ProductSpu> findOnSaleByShopId(@Param("shopId") Long shopId);

    // 统计查询
    @Query("SELECT COUNT(p) FROM ProductSpu p WHERE p.shopId = :shopId " +
           "AND p.isDeleted = 0")
    long countByShopId(@Param("shopId") Long shopId);
}
```

**规范要点**：
- 继承 `JpaRepository<Entity, ID>`
- 查询方法始终包含 `isDeleted = 0` 条件
- 复杂查询使用 `@Query` 注解
- 使用 `@Param` 绑定参数名

### 3.4 Entity 实体类

```java
@Entity
@Table(name = "product_spu")
@Data
@EqualsAndHashCode(callSuper = true)
@NoArgsConstructor
@AllArgsConstructor
public class ProductSpu extends BaseEntity {

    @Column(name = "spu_name", nullable = false, length = 200)
    private String spuName;

    @Column(name = "category_id", nullable = false)
    private Long categoryId;

    @Column(name = "brand_id")
    private Long brandId;

    @Column(name = "shop_id", nullable = false)
    private Long shopId;

    @Column(name = "min_price", precision = 10, scale = 2)
    private BigDecimal minPrice;

    @Column(name = "is_on_sale", nullable = false)
    private Integer isOnSale;

    // 关联 SKU 列表
    @OneToMany(mappedBy = "spuId", fetch = FetchType.LAZY)
    private List<ProductSku> skuList;
}
```

**规范要点**：
- 使用 `@Entity` + `@Table` 注解
- 继承 `BaseEntity`（包含 id, createTime, updateTime, isDeleted, version）
- 使用 Lombok `@Data` 简化代码
- `@Column` 显式指定数据库字段名
- 关联关系使用 LAZY 加载
- 所有实体继承统一的公共字段

## 4. 代码风格

### 4.1 缩进与空格
- 使用 4 个空格缩进（不使用 Tab）
- 运算符两侧加空格
- 方法参数之间逗号后加空格

### 4.2 行长度
- 单行代码不超过 120 字符
- 超长代码进行合理换行

### 4.3 注释规范

```java
/**
 * 商品服务接口
 * <p>
 * 提供商品的增删改查功能，支持分页查询和条件筛选
 *
 * @author OpenMall Team
 * @since 10.3.0
 */
public interface ProductService {

    /**
     * 根据 ID 查询商品
     *
     * @param id 商品 ID
     * @return 商品视图对象
     * @throws BusinessException 商品不存在时抛出
     */
    ProductVO findById(Long id);
}
```

### 4.4 空行规范
- 类成员之间空一行
- 方法之间空一行
- 逻辑块之间空一行

## 5. 异常处理

### 5.1 业务异常

```java
public class BusinessException extends RuntimeException {
    private final int code;

    public BusinessException(int code, String message) {
        super(message);
        this.code = code;
    }

    public int getCode() { return code; }
}
```

### 5.2 全局异常处理

```java
@ControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(BusinessException.class)
    @ResponseBody
    public Result<Void> handleBusinessException(BusinessException e) {
        return Result.error(e.getCode(), e.getMessage());
    }

    @ExceptionHandler(Exception.class)
    @ResponseBody
    public Result<Void> handleException(Exception e) {
        log.error("系统异常", e);
        return Result.error(500, "系统繁忙，请稍后再试");
    }
}
```

## 6. 日志规范

```java
@Slf4j
public class ProductServiceImpl implements ProductService {

    @Override
    public ProductVO findById(Long id) {
        log.debug("查询商品, id={}", id);
        ProductSpu spu = productSpuRepository.findByIdAndIsDeleted(id, 0)
                .orElseThrow(() -> new BusinessException(2001, "商品不存在"));
        log.info("商品查询成功, id={}, name={}", id, spu.getSpuName());
        return toVO(spu);
    }
}
```

**日志级别使用**：

| 级别 | 使用场景 |
|------|----------|
| ERROR | 系统错误、异常 |
| WARN | 警告信息、降级处理 |
| INFO | 关键业务流程节点 |
| DEBUG | 调试信息、详细参数 |

## 7. 安全编码规范

### 7.1 SQL 注入防护
- 所有查询使用 JPA 参数化查询
- 禁止拼接 SQL 字符串

### 7.2 XSS 防护
- 用户输入进行 HTML 转义
- 富文本使用白名单过滤

### 7.3 敏感信息处理
- 密码使用 BCrypt 加密
- 日志中不输出密码、身份证等敏感信息
- 配置文件中的密码使用环境变量或加密存储

## 8. 测试规范

```java
@SpringBootTest
class ProductServiceTest {

    @MockBean
    private ProductSpuRepository productSpuRepository;

    @Autowired
    private ProductService productService;

    @Test
    @DisplayName("查询商品 - 正常情况")
    void findById_WhenExists_ShouldReturnProduct() {
        // Given
        ProductSpu spu = new ProductSpu();
        spu.setId(1L);
        spu.setSpuName("测试商品");
        when(productSpuRepository.findByIdAndIsDeleted(1L, 0))
                .thenReturn(Optional.of(spu));

        // When
        ProductVO result = productService.findById(1L);

        // Then
        assertNotNull(result);
        assertEquals("测试商品", result.getSpuName());
    }

    @Test
    @DisplayName("查询商品 - 不存在")
    void findById_WhenNotExists_ShouldThrowException() {
        when(productSpuRepository.findByIdAndIsDeleted(1L, 0))
                .thenReturn(Optional.empty());

        assertThrows(BusinessException.class, () -> productService.findById(1L));
    }
}
```

**测试要求**：
- 单元测试覆盖率 ≥ 80%
- 使用 `@DisplayName` 描述测试场景
- 遵循 Given-When-Then 模式
