## 2. 安全概述

OpenMall 作为电商平台，安全性是首要关注点。本策略涵盖认证安全、接口安全、数据安全、运维安全等多个维度。

## 3. 认证安全

### 3.1 密码策略

| 策略 | 说明 |
|------|------|
| 加密算法 | BCrypt（Spring Security 默认） |
| 最小长度 | 8 位 |
| 复杂度 | 必须包含字母和数字 |
| 加密强度 | BCrypt 强度因子 10 |

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(10);
    }
}
```

### 3.2 登录安全

| 策略 | 说明 |
|------|------|
| 登录失败锁定 | 连续 5 次失败锁定 30 分钟 |
| 会话管理 | 基于 Redis 的分布式 Session |
| 会话超时 | 30 分钟无操作自动退出 |
| 并发登录 | 同一账号最多 3 个设备同时在线 |
| 记住我 | 支持 Remember Me，Token 有效期 7 天 |

### 3.3 权限控制

```java
// 角色定义
public enum Role {
    ROLE_BUYER("买家"),
    ROLE_MERCHANT("商家"),
    ROLE_ADMIN("管理员");

    private final String description;
}

// 权限注解
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/platform/users")
public String userList(Model model) { ... }

@PreAuthorize("hasAnyRole('MERCHANT', 'ADMIN')")
@GetMapping("/merchant/orders")
public String orderList(Model model) { ... }
```

## 4. 接口安全

### 4.1 CSRF 防护

| 策略 | 说明 |
|------|------|
| CSRF Token | 所有 POST/PUT/DELETE 请求需携带 Token |
| Thymeleaf 集成 | 表单自动注入 `_csrf` 隐藏域 |
| AJAX 集成 | 请求头携带 `X-CSRF-TOKEN` |

### 4.2 XSS 防护

| 策略 | 说明 |
|------|------|
| 输出编码 | Thymeleaf 默认 HTML 转义 |
| 输入过滤 | 富文本使用白名单过滤 |
| CSP 头 | 配置 Content-Security-Policy |

### 4.3 SQL 注入防护

| 策略 | 说明 |
|------|------|
| JPA 参数化查询 | 所有查询使用参数绑定 |
| 禁止拼接 SQL | 代码审查中严格禁止 |
| ORM 框架 | 使用 Spring Data JPA 自动参数化 |

### 4.4 接口限流

```java
@Component
public class RateLimitInterceptor implements HandlerInterceptor {

    // 登录接口：每分钟 5 次
    // 注册接口：每小时 3 次
    // 普通查询：每秒 100 次

    @Override
    public boolean preHandle(HttpServletRequest request,
                            HttpServletResponse response,
                            Object handler) {
        String key = getRateLimitKey(request);
        if (isRateLimited(key)) {
            response.setStatus(429);
            return false;
        }
        return true;
    }
}
```

## 5. 数据安全

### 5.1 敏感数据保护

| 数据类型 | 保护措施 |
|----------|----------|
| 密码 | BCrypt 单向加密 |
| 手机号 | 存储加密，展示脱敏（138****1234） |
| 身份证号 | 存储加密，展示脱敏 |
| 银行卡号 | 存储加密，展示脱敏 |
| 支付信息 | 不存储，使用第三方支付 |

### 5.2 数据访问控制

| 策略 | 说明 |
|------|------|
| 最小权限原则 | 用户只能访问自己的数据 |
| 商家数据隔离 | 商家只能查看自己店铺的数据 |
| 管理员权限分级 | 超级管理员 vs 普通管理员 |

### 5.3 逻辑删除

```java
@MappedSuperclass
@Data
public abstract class BaseEntity {
    @Column(name = "is_deleted", nullable = false)
    private Integer isDeleted = 0; // 0=正常, 1=已删除

    // 所有查询默认排除已删除数据
}

// Repository 查询示例
@Query("SELECT u FROM User u WHERE u.isDeleted = 0")
List<User> findAllActive();
```

### 5.4 乐观锁

```java
@MappedSuperclass
@Data
public abstract class BaseEntity {
    @Version
    @Column(name = "version", nullable = false)
    private Integer version = 0;
}

// JPA 自动处理版本冲突
// 当 version 不匹配时抛出 OptimisticLockException
```

## 6. 日志安全

### 6.1 审计日志

```java
@Aspect
@Component
public class AuditLogAspect {

    @Around("@annotation(auditLog)")
    public Object around(ProceedingJoinPoint point, AuditLog auditLog) {
        // 记录操作人、操作时间、操作内容、IP 地址
        log.info("AUDIT: user={}, action={}, time={}, ip={}",
                getCurrentUser(), auditLog.value(),
                LocalDateTime.now(), getClientIp());
        return point.proceed();
    }
}
```

### 6.2 敏感信息脱敏

```java
// 日志中禁止输出密码
log.info("用户登录: username={}", username); // ✓ 正确
log.info("用户登录: username={}, password={}", username, password); // ✗ 错误

// 日志中脱敏手机号
log.info("发送验证码: mobile={}", maskMobile(mobile)); // 138****1234
```

## 8. 部署安全

### 8.1 环境隔离

| 环境 | 安全级别 |
|------|----------|
| 开发环境 | 低 - 本地数据库 |
| 测试环境 | 中 - 脱敏测试数据 |
| 预发布环境 | 高 - 生产级配置 |
| 生产环境 | 最高 - 严格访问控制 |

### 8.2 安全配置

```yaml
# 生产环境配置
server:
  error:
    include-stacktrace: never
    include-message: never

spring:
  security:
    requires-secure: true

# 不暴露敏感端点
management:
  endpoints:
    web:
      exposure:
        include: health,info
```

### 8.3 Docker 安全

```dockerfile
# 使用官方镜像
FROM openjdk:21-jdk-slim

# 非 root 用户运行
RUN groupadd -r openmall && useradd -r -g openmall openmall
USER openmall

# 最小化镜像层
COPY --chown=openmall:openmall target/app.jar app.jar
ENTRYPOINT ["java", "-jar", "/app.jar"]
```

## 9. 安全事件响应

### 9.1 漏洞报告

如果您发现安全漏洞，请通过以下方式私下报告：

- 邮箱：security@openmall.com
- 不要通过公开 Issue 报告安全漏洞

### 9.2 响应流程

1. **接收报告**（24 小时内确认收到）
2. **评估漏洞**（72 小时内完成评估）
3. **制定修复方案**
4. **开发修复补丁**
5. **发布安全更新**
6. **公开披露**（修复发布后 30 天）

### 9.3 严重等级

| 等级 | 描述 | 响应时间 |
|------|------|----------|
| 严重 | 可直接获取管理员权限 | 24 小时 |
| 高危 | 可绕过认证或获取敏感数据 | 48 小时 |
| 中危 | 有限的数据泄露或功能绕过 | 7 天 |
| 低危 | 信息泄露等低风险问题 | 30 天 |

## 10. 合规要求

### 10.1 数据保护
- 遵循《个人信息保护法》
- 用户数据最小化收集
- 提供数据导出和删除功能

### 10.2 支付安全
- 遵循 PCI DSS 标准
- 不存储完整卡号
- 使用第三方支付平台
