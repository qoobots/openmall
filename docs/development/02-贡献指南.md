# OpenMall 贡献指南

## 1. 欢迎贡献

感谢您对 OpenMall 的关注！我们欢迎各种形式的贡献，包括但不限于：

- 报告 Bug
- 提交功能建议
- 改进文档
- 提交代码 PR
- 参与代码审查
- 回答社区问题

## 2. 行为准则

### 2.1 基本原则
- 尊重所有参与者
- 建设性地提出批评
- 专注于项目利益
- 耐心对待新手

### 2.2 不鼓励的行为
- 人身攻击或辱骂
- 无关的广告
- 恶意的代码提交
- 公开安全漏洞（应私下报告）

## 3. 如何贡献

### 3.1 报告 Bug

在提交 Bug 报告前，请：

1. 搜索已有 Issue，确认未被报告过
2. 使用 Bug 报告模板
3. 提供以下信息：
   - 操作系统和版本
   - JDK 版本
   - 项目版本
   - 复现步骤
   - 期望行为 vs 实际行为
   - 相关日志/截图

### 3.2 功能建议

1. 搜索已有 Issue，确认未被提议过
2. 使用功能建议模板
3. 清晰描述：
   - 功能的使用场景
   - 期望的行为
   - 可能的实现方案

### 3.3 提交代码

#### 第一步：Fork 项目

```bash
# Fork 项目到您的 GitHub 账户
# 然后克隆到本地
git clone https://github.com/YOUR_USERNAME/openmall.git
cd openmall
```

#### 第二步：创建分支

```bash
# 从 main 分支创建功能分支
git checkout -b feature/your-feature-name
```

分支命名规范：

| 前缀 | 用途 | 示例 |
|------|------|------|
| feature/ | 新功能 | feature/order-refund |
| fix/ | Bug 修复 | fix/login-redirect |
| docs/ | 文档更新 | docs/api-guide |
| refactor/ | 代码重构 | refactor/user-service |
| perf/ | 性能优化 | perf/product-query |
| test/ | 测试相关 | test/order-coverage |

#### 第三步：开发

```bash
# 编译项目
mvn clean install -DskipTests

# 运行测试
mvn test

# 运行代码检查
mvn checkstyle:check
mvn pmd:check
mvn spotbugs:check
```

#### 第四步：提交代码

```bash
git add .
git commit -m "feat: 添加订单退款功能"
git push origin feature/your-feature-name
```

### 3.4 Commit 规范

使用 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

**类型（type）**：

| 类型 | 说明 |
|------|------|
| feat | 新功能 |
| fix | Bug 修复 |
| docs | 文档更新 |
| style | 代码格式（不影响功能） |
| refactor | 重构 |
| perf | 性能优化 |
| test | 测试 |
| chore | 构建/工具变更 |
| ci | CI/CD 变更 |

**示例**：

```
feat(order): 添加订单退款功能

支持买家申请退款，商家审核退款请求

Closes #123
```

### 3.5 提交 Pull Request

1. 确保所有测试通过
2. 确保代码符合编码规范
3. 更新相关文档
4. 在 PR 描述中关联相关 Issue
5. 等待代码审查

**PR 模板**：

```markdown
## 变更类型
- [ ] 新功能
- [ ] Bug 修复
- [ ] 文档更新
- [ ] 代码重构

## 变更说明
简要描述本次变更内容

## 关联 Issue
Closes #xxx

## 测试说明
- [ ] 单元测试已通过
- [ ] 集成测试已通过
- [ ] 手动测试已完成
```

## 4. 开发环境搭建

### 4.1 环境要求

| 工具 | 版本 |
|------|------|
| JDK | 21+ |
| Maven | 4.0.0+ |
| MySQL | 8.0+ |
| Redis | 6.0+ |
| IDE | IntelliJ IDEA (推荐) |

### 4.2 快速开始

```bash
# 1. 克隆项目
git clone https://github.com/qoobot-com/openmall.git
cd openmall

# 2. 创建数据库
mysql -u root -p -e "CREATE DATABASE openmall DEFAULT CHARACTER SET utf8mb4"

# 3. 导入数据库
mysql -u root -p openmall < db/schema.sql

# 4. 编译项目
mvn clean install -DskipTests

# 5. 启动应用
cd openmall-portal && mvn spring-boot:run
cd openmall-merchant && mvn spring-boot:run
cd openmall-platform && mvn spring-boot:run
```

## 5. 代码审查标准

### 5.1 审查要点

| 类别 | 检查项 |
|------|--------|
| 功能 | 是否满足需求，边界条件是否处理 |
| 安全 | 是否有 SQL 注入、XSS、CSRF 风险 |
| 性能 | 是否有 N+1 查询，是否需要缓存 |
| 代码质量 | 是否符合编码规范，是否易于维护 |
| 测试 | 是否有充分的单元测试 |
| 文档 | 是否更新了相关文档 |

### 5.2 审查流程

1. 自动化检查（CI）通过
2. 至少一位 Maintainer 审查通过
3. 所有讨论解决
4. Maintainer 合并代码

## 6. 版本发布流程

1. 在 `develop` 分支收集变更
2. 更新 `CHANGELOG.md`
3. 更新版本号
4. 创建 Release 分支
5. 执行完整测试
6. 合并到 `main` 并打 Tag
7. 发布 Release Notes
