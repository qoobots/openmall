## 2. 部署方式概览

| 方式 | 适用场景 | 复杂度 |
|------|----------|--------|
| Maven 直接运行 | 开发测试 | 低 |
| JAR 包部署 | 单机生产 | 中 |
| Docker 部署 | 容器化环境 | 中 |
| Docker Compose | 多服务编排 | 中 |

## 3. 环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| JDK | 21+ | 运行环境 |
| MySQL | 8.0+ | 持久化存储 |
| Redis | 6.0+ | 缓存和会话 |
| Nginx | 1.20+ | 反向代理（可选） |

## 4. Maven 直接运行（开发环境）

```bash
# 1. 初始化数据库
mysql -u root -p < db/schema.sql

# 2. 编译项目
mvn clean install -DskipTests

# 3. 分别启动三个模块
cd openmall-portal && mvn spring-boot:run
cd openmall-merchant && mvn spring-boot:run
cd openmall-platform && mvn spring-boot:run
```

## 5. JAR 包部署

### 5.1 打包

```bash
mvn clean package -DskipTests

# 生成的 JAR 包
# openmall-portal/target/openmall-portal-10.3.0-SNAPSHOT.jar
# openmall-merchant/target/openmall-merchant-10.3.0-SNAPSHOT.jar
# openmall-platform/target/openmall-platform-10.3.0-SNAPSHOT.jar
```

### 5.2 运行

```bash
# 前台商城 (端口 8081)
nohup java -jar openmall-portal-10.3.0-SNAPSHOT.jar \
    --server.port=8081 --spring.profiles.active=prod > portal.log 2>&1 &

# 商家后台 (端口 8082)
nohup java -jar openmall-merchant-10.3.0-SNAPSHOT.jar \
    --server.port=8082 --spring.profiles.active=prod > merchant.log 2>&1 &

# 平台管理 (端口 8083)
nohup java -jar openmall-platform-10.3.0-SNAPSHOT.jar \
    --server.port=8083 --spring.profiles.active=prod > platform.log 2>&1 &
```

### 5.3 JVM 参数建议

```bash
java -jar \
    -Xms512m -Xmx1024m \
    -XX:+UseZGC \
    -XX:+ZGenerational \
    -XX:MaxGCPauseMillis=200 \
    -Dspring.threads.virtual.enabled=true \
    -Dfile.encoding=UTF-8 \
    openmall-portal-10.3.0-SNAPSHOT.jar
```

## 6. Docker 部署

### 6.1 Dockerfile

```dockerfile
FROM openjdk:21-jdk-slim
ENV TZ=Asia/Shanghai
RUN ln -snf /usr/share/zoneinfo/$TZ /etc/localtime && echo $TZ > /etc/timezone
WORKDIR /app
COPY target/openmall-portal-10.3.0-SNAPSHOT.jar app.jar
RUN groupadd -r openmall && useradd -r -g openmall openmall
USER openmall
EXPOSE 8081
ENTRYPOINT ["java", "-XX:+UseZGC", "-XX:+ZGenerational", \
    "-Dspring.threads.virtual.enabled=true", "-jar", "app.jar"]
```

### 6.2 构建和运行

```bash
docker build -t openmall-portal:10.3.0 .
docker run -d --name openmall-portal -p 8081:8081 \
    -e SPRING_DATASOURCE_URL=jdbc:mysql://host.docker.internal:3306/openmall \
    -e SPRING_DATASOURCE_USERNAME=root \
    -e SPRING_DATASOURCE_PASSWORD=your_password \
    -e SPRING_DATA_REDIS_HOST=host.docker.internal \
    openmall-portal:10.3.0
```

## 7. Docker Compose 部署

### 7.1 docker-compose.yml

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: openmall-mysql
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: openmall
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - mysql-data:/var/lib/mysql
      - ../db/schema.sql:/docker-entrypoint-initdb.d/schema.sql
    command: --character-set-server=utf8mb4 --collation-server=utf8mb4_unicode_ci
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: openmall-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    command: redis-server --appendonly yes
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  portal:
    build:
      context: ../openmall-portal
      dockerfile: ../docker/Dockerfile.portal
    container_name: openmall-portal
    ports:
      - "8081:8081"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/openmall
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root123
      SPRING_DATA_REDIS_HOST: redis
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }

  merchant:
    build:
      context: ../openmall-merchant
      dockerfile: ../docker/Dockerfile.merchant
    container_name: openmall-merchant
    ports:
      - "8082:8082"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/openmall
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root123
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_DATABASE: 1
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }

  platform:
    build:
      context: ../openmall-platform
      dockerfile: ../docker/Dockerfile.platform
    container_name: openmall-platform
    ports:
      - "8083:8083"
    environment:
      SPRING_DATASOURCE_URL: jdbc:mysql://mysql:3306/openmall
      SPRING_DATASOURCE_USERNAME: root
      SPRING_DATASOURCE_PASSWORD: root123
      SPRING_DATA_REDIS_HOST: redis
      SPRING_DATA_REDIS_DATABASE: 2
    depends_on:
      mysql: { condition: service_healthy }
      redis: { condition: service_healthy }

  nginx:
    image: nginx:1.25-alpine
    container_name: openmall-nginx
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - portal
      - merchant
      - platform

volumes:
  mysql-data:
  redis-data:
```

### 7.2 Nginx 配置

```nginx
upstream portal { server portal:8081; }
upstream merchant { server merchant:8082; }
upstream platform { server platform:8083; }

server {
    listen 80;
    server_name mall.example.com;
    location / {
        proxy_pass http://portal;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /merchant {
        proxy_pass http://merchant;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
    location /platform {
        proxy_pass http://platform;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}
```

### 7.3 启动

```bash
docker-compose up -d          # 启动所有服务
docker-compose logs -f        # 查看日志
docker-compose down           # 停止服务
```

## 10. 常见部署问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据库连接失败 | 配置错误或服务未启动 | 检查 MySQL 服务和连接配置 |
| Redis 连接失败 | Redis 未启动 | 启动 Redis 服务 |
| 端口被占用 | 端口冲突 | 修改端口或停止占用进程 |
| 内存溢出 | JVM 内存不足 | 增大 -Xmx 参数 |
| 静态资源 404 | 路径配置问题 | 检查静态资源目录配置 |
