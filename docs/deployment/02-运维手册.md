## 2. 日常运维

### 2.1 服务管理

```bash
# 启动/停止/重启
systemctl start openmall-portal
systemctl stop openmall-portal
systemctl restart openmall-portal
systemctl status openmall-portal

# 查看日志
journalctl -u openmall-portal -f
tail -f /var/log/openmall/portal.log
```

### 2.2 Docker 管理

```bash
docker ps                                    # 查看运行容器
docker logs -f openmall-portal               # 查看日志
docker exec -it openmall-portal bash         # 进入容器
docker restart openmall-portal               # 重启
docker stats                                 # 资源使用
```

## 3. 数据库运维

### 3.1 备份

```bash
# 全量备份
mysqldump -u root -p openmall > backup_$(date +%Y%m%d).sql

# 压缩备份
mysqldump -u root -p openmall | gzip > backup_$(date +%Y%m%d).sql.gz

# 定时备份 (crontab)
0 2 * * * mysqldump -u root -p'password' openmall | gzip > /backup/openmall_$(date +\%Y\%m\%d).sql.gz
```

### 3.2 恢复

```bash
mysql -u root -p openmall < backup_20260630.sql
gunzip < backup_20260630.sql.gz | mysql -u root -p openmall
```

### 3.3 日常维护

```sql
-- 查看表大小
SELECT table_name,
       ROUND(((data_length + index_length) / 1024 / 1024), 2) AS size_mb
FROM information_schema.tables
WHERE table_schema = 'openmall' ORDER BY size_mb DESC;

-- 分析表
ANALYZE TABLE product_spu, order_master, user;

-- 清理过期会话
DELETE FROM spring_session WHERE last_access_time < DATE_SUB(NOW(), INTERVAL 7 DAY);
```

## 4. Redis 运维

```bash
redis-cli -h localhost -p 6379
INFO                    # 查看信息
INFO memory             # 查看内存
DBSIZE                  # 查看 key 数量
SLOWLOG GET 10          # 查看慢查询
```

**监控阈值**：

| 指标 | 阈值 | 说明 |
|------|------|------|
| 内存使用率 | < 80% | 超过需扩容或清理 |
| 命中率 | > 90% | 低于需优化缓存策略 |
| 连接数 | < 1000 | 超过需检查连接泄漏 |

## 5. 日志管理

### 5.1 日志文件

| 文件 | 路径 | 说明 |
|------|------|------|
| 应用日志 | /var/log/openmall/portal.log | 业务日志 |
| 访问日志 | /var/log/openmall/access.log | HTTP 访问日志 |
| GC 日志 | /var/log/openmall/gc.log | 垃圾回收日志 |

### 5.2 日志轮转

```bash
# /etc/logrotate.d/openmall
/var/log/openmall/*.log {
    daily
    rotate 30
    compress
    delaycompress
    missingok
    notifempty
    copytruncate
    dateext
    dateformat -%Y%m%d
}
```

### 5.3 日志查看

```bash
tail -f /var/log/openmall/portal.log
grep "ERROR" /var/log/openmall/portal.log
grep "ERROR" /var/log/openmall/portal.log | awk '{print $5}' | sort | uniq -c | sort -rn
```

## 6. 性能监控

### 6.1 系统资源

```bash
top -b -n 1 | grep java    # CPU
free -h                     # 内存
df -h                       # 磁盘
netstat -an | grep ESTABLISHED | wc -l  # 连接数
```

### 6.2 JVM 监控

```bash
jps -l                      # 查看 JVM 进程
jstat -gc <pid> 1000        # 堆内存
jstack <pid>                # 线程
jmap -dump:format=b,file=heap.hprof <pid>  # 堆转储
```

### 6.3 应用指标

| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| QPS | 每秒请求数 | 根据容量设定 |
| 响应时间 | P99 响应时间 | P99 > 2s |
| 错误率 | 5xx 错误占比 | > 1% |
| 数据库连接 | 活跃连接数 | > 80% 最大连接数 |

## 7. 故障处理

### 7.1 服务不可用

```bash
systemctl status openmall-portal
curl http://localhost:8081/actuator/health
tail -100 /var/log/openmall/portal.log | grep ERROR
mysql -u root -p -e "SELECT 1"
redis-cli ping
systemctl restart openmall-portal
```

### 7.2 数据库连接池耗尽

```bash
mysql -u root -p -e "SHOW PROCESSLIST"
mysql -u root -p -e "SHOW FULL PROCESSLIST" | grep -v Sleep
mysql -u root -p -e "KILL <connection_id>"
```

### 7.3 内存溢出

```bash
jmap -dump:format=b,file=/tmp/heap.hprof <pid>
# 使用 MAT 或 jhat 分析
systemctl restart openmall-portal
```

### 7.4 磁盘空间不足

```bash
find / -type f -size +100M -exec ls -lh {} \; 2>/dev/null
find /var/log/openmall -name "*.log" -mtime +30 -delete
docker system prune -a -f
```

## 8. 应急预案

### 8.1 数据库故障
1. 确认主库故障 → 2. 切换应用到从库 → 3. 修改连接配置 → 4. 重启应用 → 5. 恢复主库后切回

### 8.2 Redis 故障
1. 确认 Redis 故障 → 2. 应用自动降级 → 3. 重启 Redis → 4. 预热缓存 → 5. 恢复

### 8.3 服务雪崩
1. 启动限流 → 2. 关闭非核心功能 → 3. 扩容实例 → 4. 分析根因 → 5. 逐步恢复
