## 2. 项目概述

OpenMall 是一个基于 Spring Boot 3.5.10 + JDK 21 + Thymeleaf + MySQL 的 B2B2C 开源电商平台系统。项目采用前后端不分离架构，使用 Thymeleaf 模板引擎渲染页面，模仿天猫商城的 B2B2C 模式，支持买家端（前台商城）、卖家端（商家后台）和管理端（平台管理）三大业务场景。

## 3. 设计目标与原则

### 3.1 设计目标

| 目标 | 说明 |
|------|------|
| 高性能 | 利用 JDK 21 Virtual Threads 提升并发处理能力 |
| 高可用 | 模块化部署，支持横向扩展 |
| 易扩展 | 清晰的模块边界，预留支付/物流/搜索等接口 |
| 安全性 | 多层安全防护，满足电商平台安全要求 |
| 可维护 | 规范的代码结构，完善的文档体系 |

### 3.2 设计原则

- **SOLID 原则**：单一职责、开闭原则、里氏替换、接口隔离、依赖倒置
- **分层架构**：Controller → Service → Repository 严格分层
- **领域驱动**：核心业务逻辑集中在 Domain 层
- **配置外部化**：所有环境相关配置通过 application.yml 管理

### 4.2 模块划分

#### openmall-common（公共模块）
```
openmall-common/
├── common-core/          # 核心工具类、常量、枚举
│   ├── constant/         # 常量定义 (RedisConstants 等)
│   ├── enums/            # 枚举类 (UserType, OrderStatus, PayType 等)
│   └── result/           # 统一响应结果 (Result, PageResult)
├── common-domain/        # 领域对象
│   └── entity/           # JPA 实体类 (BaseEntity, User, ProductSpu 等)
├── common-security/      # 安全认证模块
└── common-web/           # Web 通用配置
```

#### openmall-portal（前台商城 - 买家端）
```
openmall-portal/
├── controller/           # 控制器 (IndexController, AuthController, ProductController 等)
├── service/              # 服务层 (UserService, ProductService, OrderService 等)
├── repository/           # 数据访问 (UserRepository, ProductSpuRepository 等)
├── dto/                  # 数据传输对象
├── vo/                   # 视图对象
├── config/               # 配置类
└── resources/
    ├── templates/        # Thymeleaf 模板
    └── static/           # 静态资源 (CSS, JS, 图片)
```

#### openmall-merchant（商家后台 - 卖家端）
```
openmall-merchant/

├── repository/           # 数据访问
└── resources/
    ├── templates/        # 商家后台 Thymeleaf 模板
    └── static/           # 静态资源
```

#### openmall-platform（平台管理 - 管理员端）
```
openmall-platform/
```

## 7. 核心设计模式

### 7.1 MVC 模式
严格遵循 Model-View-Controller 模式，Controller 接收请求、Service 处理业务、Repository 操作数据、Thymeleaf 模板渲染视图。

### 7.2 模板方法模式
`BaseEntity` 作为所有实体的基类，提供 `id`、`createTime`、`updateTime`、`isDeleted`、`version` 等公共字段。

### 7.3 仓储模式
通过 Spring Data JPA Repository 接口封装数据访问逻辑。

### 7.4 统一响应
所有 API 返回使用 `Result<T>` 和 `PageResult<T>` 统一包装。

## 8. 安全架构

### 8.1 认证安全
- 密码 BCrypt 加密存储
- Session 管理
- 记住我（Remember Me）功能
- 登录失败锁定

### 8.2 接口安全
- CSRF 防护
- XSS 防护
- SQL 注入防护（JPA 参数化查询）

### 8.3 数据安全
- 敏感信息脱敏
- 逻辑删除（isDeleted）
- 乐观锁（version 字段）

## 9. 性能优化策略

| 策略 | 实现方式 |
|------|----------|
| 虚拟线程 | JDK 21 Virtual Threads 提升并发 |
| 数据库连接池 | HikariCP 高效连接管理 |
| Redis 缓存 | 热点数据缓存、会话管理 |
| 分页查询 | Spring Data JPA Pageable |
| 数据库索引 | 常用查询字段建立索引 |
