## 2. 数据库概述

OpenMall 使用 MySQL 8.0+ 作为持久化存储，采用 utf8mb4 字符集以支持 Emoji 等多字节字符。核心业务表采用逻辑删除和乐观锁机制，确保数据安全和并发安全。

## 4. 核心表结构设计

### 4.1 用户表 (user)

|--------|------|------|--------|--------|------|

| birthday | VARCHAR | 20 | Y | - | 生日 |
| user_type | INT | - | N | 1 | 用户类型：1=买家,2=商家,3=管理员 |
| status | INT | - | N | 0 | 状态：0=正常,1=冻结,2=禁用 |
| last_login_time | DATETIME | - | Y | - | 最后登录时间 |
| last_login_ip | VARCHAR | 50 | Y | - | 最后登录 IP |

- PRIMARY KEY: `id`

## 6. 索引设计策略

| 策略 | 说明 |
|------|------|
| 主键索引 | 所有表使用自增 BIGINT 作为主键 |
| 唯一索引 | 用户名、订单编号等业务唯一字段 |

| 联合索引 | 高频联合查询条件字段组合 |

## 7. 数据库规范

### 7.1 命名规范
- 表名：小写下划线命名法，单数形式（如 `user`、`product_spu`）
- 字段名：小写下划线命名法（如 `user_name`、`order_status`）
- 索引名：`uk_` 前缀表示唯一索引，`idx_` 前缀表示普通索引

### 7.2 数据类型规范
- 主键：BIGINT
- 字符串：VARCHAR（不定长）、TEXT（长文本）
- 金额：DECIMAL(10,2)
- 状态/标记：INT
- 时间：DATETIME

### 7.3 安全规范
- 密码使用 BCrypt 加密存储
- 敏感信息字段脱敏
- 逻辑删除替代物理删除
- 乐观锁防止并发更新冲突
