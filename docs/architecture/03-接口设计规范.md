## 2. 统一响应格式

### 2.1 成功响应

```json
{
    "code": 200,
    "message": "操作成功",
    "data": { ... }
}
```

### 2.2 分页响应

```json
{
    "code": 200,
    "message": "操作成功",
    "data": {
        "records": [ ... ],
        "total": 100,
        "size": 10,
        "current": 1,
        "pages": 10
    }

```

### 2.3 错误响应

```json
{
    "code": 500,
    "message": "系统错误",
    "data": null

```

## 3. HTTP 状态码规范

| 状态码 | 含义 | 使用场景 |
|--------|------|----------|
| 200 | 成功 | 请求处理成功 |
| 201 | 已创建 | 资源创建成功 |
| 302 | 重定向 | 页面跳转 |
| 400 | 请求错误 | 参数校验失败 |
| 401 | 未认证 | 未登录或 Token 过期 |
| 403 | 无权限 | 权限不足 |
| 404 | 未找到 | 资源不存在 |
| 500 | 服务器错误 | 系统内部异常 |

## 5. URL 命名规范

### 5.1 基础规范

```
HTTP 方法 + URL 路径 → 操作语义

GET     /api/products          → 查询商品列表
GET     /api/products/{id}     → 查询单个商品
POST    /api/products          → 新增商品
PUT     /api/products/{id}     → 更新商品
DELETE  /api/products/{id}     → 删除商品
```

### 5.2 模块前缀

| 模块 | 前缀 | 说明 |
|------|------|------|
| 前台商城 | / | 买家端，无特殊前缀 |
| 商家后台 | /merchant | 卖家端 API |
| 平台管理 | /platform | 管理员端 API |

## 7. 参数规范

### 7.1 分页参数

```
page:   当前页码（从 1 开始）
size:   每页条数（默认 10）
sort:   排序字段
order:  排序方向（asc/desc）
```

示例：`/products?page=1&size=20&sort=createTime&order=desc`

### 7.2 搜索参数

```
keyword:    关键词搜索
categoryId: 分类筛选
brandId:    品牌筛选
minPrice:   最低价格
maxPrice:   最高价格
```

示例：`/products?keyword=手机&categoryId=1&minPrice=1000&maxPrice=5000`

## 8. 安全规范

### 8.1 认证方式
- 使用 Spring Security Session 认证
- 登录成功后 Session 存储用户信息
- 敏感操作需二次验证

### 8.2 CSRF 防护
- 所有 POST/PUT/DELETE 请求需携带 CSRF Token
- Thymeleaf 模板自动注入 CSRF Token

### 8.3 XSS 防护
- 所有用户输入在服务端进行 HTML 转义
- 富文本内容使用白名单过滤

### 8.4 接口限流
- 登录接口：每分钟 5 次
- 注册接口：每小时 3 次
- 普通查询接口：每秒 100 次
